# SOC 2 Access Control Audit Project

## Project Overview

This project simulates a Junior GRC Analyst assignment for a SaaS company preparing for a SOC 2 audit. The goal is to review access management controls, identify evidence requirements, test sample controls, document control gaps, assess risk, and recommend remediation actions.

## Scenario

The mock company is a 120-employee SaaS organization using common business and cloud systems including an identity provider, cloud infrastructure, source code repository, collaboration tools, and HR system.

The company is preparing for a SOC 2 Type II audit and needs to validate whether access controls are properly designed, documented, and operating effectively.

## Objectives

- Identify audit evidence needed for access control testing
- Test user access review and offboarding controls
- Document control gaps and risk ratings
- Create a remediation tracker for identified findings
- Build interview-ready GRC project experience
