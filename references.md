# References

## Purpose

This document lists frameworks, concepts, and resources connected to the SOC 2 Access Control Audit Project.

The project is a mock GRC portfolio project and does not use confidential company data.

---

## Frameworks and Standards Referenced

### SOC 2 Trust Services Criteria

SOC 2 is commonly used by service organizations to demonstrate controls related to security, availability, processing integrity, confidentiality, and privacy.

This project focuses mainly on access control concepts related to the Security category.

Relevant areas include:

- Logical access controls
- User access provisioning
- User access removal
- Privileged access
- Authentication
- Change and access approval
- Monitoring and evidence retention

---

### NIST Cybersecurity Framework

The NIST Cybersecurity Framework helps organizations identify, protect, detect, respond, and recover from cybersecurity risks.

This project connects most closely to:

- Identify: asset and access ownership
- Protect: identity management and access control
- Detect: monitoring unauthorized or inappropriate access
- Respond: documenting and remediating findings

---

### NIST SP 800-53 Concepts

NIST SP 800-53 includes security and privacy controls used by many organizations.

This project relates to control families such as:

- AC: Access Control
- IA: Identification and Authentication
- AU: Audit and Accountability
- CA: Assessment, Authorization, and Monitoring
- CM: Configuration Management
- RA: Risk Assessment

---

### ISO/IEC 27001 Concepts

ISO/IEC 27001 focuses on information security management systems.

This project relates to concepts such as:

- Access control
- User access management
- Privileged access management
- Supplier/vendor security
- Risk assessment
- Corrective actions
- Evidence and documentation
